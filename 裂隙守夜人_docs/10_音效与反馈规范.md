# 音效与战斗反馈规范

## 1. 音效分类

- UI 音效
- 玩家动作
- 武器攻击
- 技能
- 命中
- 敌人
- Boss
- 环境
- 音乐

## 2. 必须音效

玩家：

- 跑步
- 跳跃
- 落地
- 三段攻击
- 闪避
- 受击
- 死亡

技能：

- 烈焰剑气释放
- 火焰爆炸
- 雷影冲刺
- 连锁雷电
- 旋刃风暴
- 大招蓄力
- 大招命中

敌人：

- 出现
- 攻击
- 受击
- 死亡
- 精英警告
- Boss 登场

UI：

- 按钮悬停
- 确认
- 取消
- 升级出现
- 选择卡片
- 结算

## 3. 音频总线

建议：

- Master
- Music
- SFX
- UI
- Ambience

设置界面分别控制音量。

## 4. 同类音效限制

大量敌人同时受击时，不能播放几十个完全相同的音效。

需要：

- 并发上限
- 随机音高
- 随机音量
- 同类音效最小播放间隔
- 根据距离降低音量

## 5. 视觉反馈优先级

玩家必须清楚感知：

- 自己是否命中
- 自己是否受伤
- 技能是否释放成功
- 技能是否冷却完毕
- Boss 是否进入新阶段
- 是否触发完美闪避
- 是否完成技能进化

## 6. 屏幕震动等级

- 轻：普通攻击第三段
- 中：重击、精英死亡
- 强：Boss 攻击、大招
- 极强：仅用于剧情或最终击杀

设置中允许调整为：

- 关闭
- 低
- 标准
- 高
