# 开发阶段与里程碑

## Phase 0：项目初始化

目标：

- 创建 Godot 项目
- 建立目录
- 配置分辨率
- 配置输入
- 创建基础场景
- 建立 Git 忽略文件
- 创建全局单例骨架

验收：

- 项目可以启动
- 可以进入空白战斗场景
- 无解析错误

## Phase 1：玩家移动原型

实现：

- 左右移动
- 跳跃
- 下落
- 重力
- 土狼时间
- 输入缓冲
- 朝向
- 摄像机跟随
- 简单平台

验收：

- 移动顺畅
- 跳跃不粘连
- 摄像机不抖动

## Phase 2：普通攻击和受击

实现：

- 三段连击
- Hitbox/Hurtbox
- 伤害
- 击退
- 敌人受击
- 玩家受击
- 死亡
- 命中停顿
- 基础震动

验收：

- 三段攻击可稳定连出
- 同一次攻击不会错误重复伤害
- 死亡状态不会恢复移动

## Phase 3：闪避和完美闪避

实现：

- 闪避位移
- 无敌帧
- 冷却
- 完美闪避
- 残影
- UI 提示

验收：

- 闪避能穿过攻击
- 完美闪避只触发一次
- 不能无限连续闪避

## Phase 4：敌人框架

实现：

- 敌人状态机
- 近战杂兵
- 弓箭手
- 自爆怪
- 飞行魔眼
- 敌人死亡
- 经验掉落

验收：

- 各敌人行为不同
- 敌人不会卡死
- 死亡后正确回收

## Phase 5：波次和涌兵

实现：

- WaveData
- WaveManager
- 敌人权重
- 最大存活数
- 难度倍率
- 精英出现
- 对象池

验收：

- 10分钟连续运行稳定
- 敌人数量逐步增加
- 不会在玩家附近突然生成

## Phase 6：经验和升级

实现：

- 经验球
- 吸附
- 等级
- 升级暂停
- 三选一
- 被动升级
- 重抽
- 候选过滤

验收：

- 多次升级不丢失
- 满级项目不再出现
- UI 恢复后游戏继续

## Phase 7：主动技能

实现：

- 技能基类
- 冷却组件
- 烈焰剑气
- 雷影冲刺
- 旋刃风暴
- 技能 HUD

验收：

- 三个技能可独立释放
- 冷却正确
- 技能不依赖玩家脚本硬编码

## Phase 8：技能进化

实现：

- 技能等级
- 分支选择
- 最终进化
- 分支互斥
- 技能行为变化
- 技能协同标签

验收：

- 每个技能至少两条路线
- 最终形态有明显视觉和玩法变化

## Phase 9：大招

实现：

- 大招能量
- 能量获取
- 天陨斩
- 无敌
- 镜头演出
- 大招 HUD

验收：

- 能量满后可释放
- 大招不能重复触发
- 演出结束后状态恢复

## Phase 10：Boss

实现：

- Boss 状态机
- 三阶段
- 韧性
- Boss 血条
- 阶段转换
- 胜利流程

验收：

- Boss 拥有至少3种攻击
- 阶段转换稳定
- 击败后进入结算

## Phase 11：完整流程

实现：

- 主菜单
- 角色选择
- 战斗
- 暂停
- 失败
- 胜利
- 结算
- 重新开始
- 返回菜单

验收：

- 可连续游玩多局
- 场景切换后状态正确重置

## Phase 12：局外成长

实现：

- 裂隙结晶
- 解锁技能
- 简化天赋树
- 存档
- 设置保存

验收：

- 关闭游戏后进度保留
- 存档损坏时能够使用默认数据

## Phase 13：优化与发布

实现：

- 性能测试
- 对象池优化
- 资源检查
- 输入适配
- Windows 导出
- Android 横屏初步适配

验收：

- 目标设备稳定运行
- 无阻断性错误
