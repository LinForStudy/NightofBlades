# UI / UX 设计规范

## 1. 视觉方向

- 像素风，但 UI 不必完全低分辨率
- 使用深色半透明面板
- 强调金色、红色、蓝色等状态色
- 图标轮廓清晰
- 战斗区域优先，UI 不遮挡角色

## 2. 设计分辨率

- 1280×720
- 采用 stretch mode canvas_items
- 保持 16:9 横屏
- UI 使用安全边距
- 后续移动端按钮点击区域不小于72×72逻辑像素

## 3. 战斗 HUD

### 左上角

- 头像
- 生命条
- 护盾条
- 等级
- 经验条

### 右上角

- 时间
- 击杀数
- 连击数
- 当前波次或危险等级

### 下方中央

- 技能1
- 技能2
- 技能3
- 大招

技能按钮显示：

- 图标
- 快捷键
- 冷却遮罩
- 剩余秒数
- 当前等级
- 分支标记

### Boss 出现时

顶部中央显示：

- Boss 名称
- Boss 血条
- 阶段标记

## 4. 升级卡片

升级界面一次显示三张卡片。

卡片尺寸建议：

- 280×380
- 中间卡片略微放大
- 鼠标悬停时高亮
- 键盘左右选择
- 确认键选择

卡片内容顺序：

1. 稀有度
2. 图标
3. 名称
4. 当前等级→新等级
5. 核心效果
6. 数值变化
7. 流派标签

文案必须具体。

错误示例：

“技能变得更强。”

正确示例：

“剑气宽度增加100%，穿透上限移除，但飞行速度降低25%。”

## 5. 主菜单

主菜单背景使用战场动态场景或像素插画。

菜单项：

- 开始游戏
- 角色配置
- 天赋
- 图鉴
- 设置
- 退出

MVP 中不可用功能可以隐藏，而不是显示大量“敬请期待”。

## 6. 暂停界面

包含：

- 继续
- 操作说明
- 设置
- 重新开始
- 返回主菜单

暂停时：

- 游戏逻辑停止
- UI 动画可以继续
- 音频降低音量

## 7. 结算界面

分为胜利和失败两种状态。

必须展示：

- 存活时间
- 击杀数
- 玩家等级
- 总伤害
- 最高连击
- 主要技能伤害占比
- 获得资源

## 8. 可访问性

设置中至少包含：

- 主音量
- 音乐音量
- 音效音量
- 屏幕震动强度
- 伤害数字开关
- 闪烁效果强度
- 全屏
- 分辨率

## 9. 验收标准

- 1280×720 下所有信息完整
- UI 不覆盖玩家主要活动区域
- 键盘和鼠标均可操作菜单
- 升级描述清楚
- 技能冷却状态一眼可见
