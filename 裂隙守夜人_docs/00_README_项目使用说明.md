# 《裂隙守夜人》项目文档包

## 1. 项目目标

本项目是一款使用 Godot 4.x 开发的：

- 2D 横版像素动作游戏
- Roguelite 局内构筑
- 持续涌兵生存
- 主动操作战斗
- 支持平A、闪避、3个技能和1个大招
- 技能能够升级、分支和最终进化

第一阶段目标不是一次性完成完整商业游戏，而是先完成一个能够验证核心乐趣的可玩版本。

## 2. 推荐开发顺序

Codex 必须严格按照以下顺序开发：

1. 项目基础与目录结构
2. 玩家移动、跳跃、闪避
3. 普通攻击与受击系统
4. 敌人基础 AI
5. 涌兵和波次系统
6. 经验与升级系统
7. 主动技能框架
8. 技能进化系统
9. Boss 战
10. UI 和游戏流程
11. 音效、特效和打击感
12. 存档与局外成长
13. 性能优化与打包

禁止在核心战斗没有完成前，提前制作复杂商城、装备、剧情、多人联机或大量地图。

## 3. 文档使用顺序

建议 Codex 每次只处理一个阶段。

开始任务前，先阅读：

- `01_PRD_产品需求文档.md`
- `02_GDD_游戏设计文档.md`
- `11_技术架构设计.md`
- 当前阶段对应的专项文档
- `15_Codex分阶段执行提示词.md`

完成每个阶段后必须：

- 运行项目
- 检查报错
- 输出修改文件清单
- 输出实现结果
- 输出未完成项
- 保存运行截图
- 不得在存在解析错误时进入下一阶段

## 4. 推荐技术栈

- 引擎：Godot 4.4 或当前稳定版 Godot 4.x
- 脚本：GDScript
- 目标平台：Windows 优先，后续支持 Android 横屏
- 设计分辨率：1280×720
- 像素素材基准：角色 48×48 或 64×64
- 版本管理：Git
- 数据格式：Godot Resource + JSON 存档

## 5. MVP 完成定义

满足以下条件才算 MVP 完成：

- 玩家可以移动、跳跃、闪避
- 有三段平A
- 有受击、死亡和无敌帧
- 至少有4种普通敌人
- 敌人会持续生成并逐渐变强
- 玩家可以获得经验并升级
- 至少有3个主动技能
- 每个技能至少有两条进化路线
- 有1个大招
- 有1个精英敌人和1个最终 Boss
- 有完整的开始、战斗、暂停、结算流程
- 可以完成一局 10～15 分钟的游戏
- 没有阻断流程的严重错误

## 6. 核心原则

1. 优先保证动作手感。
2. 所有系统必须数据驱动。
3. 角色、技能和敌人之间使用信号或事件解耦。
4. 不允许把所有代码写进一个脚本。
5. 每一阶段都必须可以独立运行和验收。
6. 先使用占位素材完成玩法，再替换正式美术。

## 7. Agent Sprite Forge 美术管线

本项目使用 `agent-sprite-forge` 作为 Codex 美术资产生成与后处理管线。

开始制作正式美术前，请阅读：

- `18_AgentSpriteForge美术生产管线.md`
- `19_Codex美术资产执行提示词.md`

美术生产采用“风格锁定 → 主角基础动作 → 平A与特效 → 敌人 → 主动技能 → 地图 → Boss”的批次方式，不允许一次生成全部资产。
