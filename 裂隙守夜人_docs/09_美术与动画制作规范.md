# 美术与动画制作规范

## 1. 总体风格

精细像素风，重点表现：

- 清晰轮廓
- 强对比动作
- 夸张武器
- 明确的受击反馈
- 有层次的技能特效

不建议采用过于细碎、低对比度的写实像素风。

## 2. 素材规格

### 主角

建议画布：

- 单帧 64×64
- 实际人物高度约48～56像素
- 武器可以超出身体范围

### 普通敌人

- 小型：32×32或48×48
- 中型：64×64
- 大型：96×96以上

### Boss

- 128×128至192×192
- 根据攻击动作允许超出基础画布

### 技能图标

- 原始尺寸64×64
- UI 中可缩放到48×48或56×56

## 3. 主角动画清单

MVP 必须：

- idle：6帧
- run：8帧
- jump_start：2帧
- jump_loop：2帧
- fall：2帧
- attack_1：6帧
- attack_2：6帧
- attack_3：8帧
- air_attack：6帧
- charge：4帧循环
- charge_release：8帧
- dodge：6帧
- hurt：3帧
- death：8帧
- skill_fire：8帧
- skill_dash：6帧
- skill_spin：8帧循环
- ultimate：12帧以上

## 4. 动画事件

动画中通过 Call Method Track 或 AnimationPlayer 事件触发：

- 开启 Hitbox
- 关闭 Hitbox
- 生成剑气
- 播放脚步音
- 生成残影
- 触发震动
- 结束技能状态

不允许仅根据固定计时器猜测攻击帧。

## 5. 特效层级

推荐绘制层级：

1. 远景
2. 地图主体
3. 地面装饰
4. 敌人
5. 玩家
6. 前景技能特效
7. 伤害数字
8. UI

## 6. 打击感特效

普通攻击：

- 刀光
- 小型火花
- 敌人闪白
- 轻微残影

重攻击：

- 更大的刀光
- 冲击环
- 屏幕震动
- 更长停顿

大招：

- 画面压暗
- 聚光
- 屏幕缩放
- 大范围像素粒子
- 强烈音效
- 短暂慢动作

## 7. 素材命名

统一英文小写蛇形命名：

- player_idle.png
- player_attack_01.png
- enemy_grunt_run.png
- skill_fire_wave.png
- icon_skill_fire_wave.png
- fx_hit_small.png

## 8. 导入设置

像素素材：

- Filter：关闭
- Mipmaps：关闭
- Repeat：根据用途
- 压缩：无损
- 使用整数缩放
- 避免非整数比例造成模糊

## 9. 占位素材策略

开发前期允许使用：

- 纯色矩形
- 简单像素轮廓
- 临时图标
- 免费占位音效

但所有占位素材必须放入：

`assets/_placeholder/`

后续替换时不改代码路径，或者通过 Resource 指向正式素材。
