# 《裂隙守夜人》产品需求文档 PRD

## 1. 产品概述

### 1.1 产品名称

暂定名：《裂隙守夜人》

### 1.2 产品类型

2D 横版像素动作 Roguelite 生存游戏。

### 1.3 核心用户

- 喜欢横版动作游戏的玩家
- 喜欢 Roguelite 构筑的玩家
- 喜欢割草、生存和成长反馈的玩家
- 喜欢 10～20 分钟单局体验的轻中度玩家

### 1.4 产品卖点

- 不是自动攻击，而是主动平A、闪避和释放技能
- 技能升级会改变形态，而不是只增加数字
- 敌人持续涌入，后期形成大规模战斗
- 同一个主角可以构筑火焰、雷影、分身、冰冻、狂战等流派
- 一局内角色从普通剑士成长为满屏技能的强大战士

## 2. 核心玩法循环

### 2.1 局内循环

移动与攻击  
→ 击败敌人  
→ 获得经验  
→ 升级三选一  
→ 强化技能或属性  
→ 面对更高强度敌人  
→ 击败精英或 Boss  
→ 获得结算奖励

### 2.2 局外循环

完成一局  
→ 获得裂隙结晶  
→ 解锁新技能、武器、天赋  
→ 调整开局配置  
→ 挑战更高难度

## 3. 游戏模式

### 3.1 标准生存模式

- 单局时长：12～15 分钟
- 最终目标：击败最终 Boss
- 包含普通波次、精英波次和 Boss 波次

### 3.2 无限模式

- 完成标准模式后解锁
- 敌人无限增强
- 记录最高时间、最高等级和最高击杀数

### 3.3 挑战模式

后续版本实现：

- 禁止回血
- 只能使用近战
- 技能冷却加倍
- 敌人速度提升
- 固定技能组合

## 4. 战斗功能需求

### 4.1 玩家能力

玩家必须支持：

- 左右移动
- 跳跃
- 下落
- 闪避
- 三段普通攻击
- 空中攻击
- 蓄力攻击
- 三个主动技能
- 一个大招
- 受击
- 击退
- 无敌帧
- 死亡

### 4.2 战斗目标

- 普通战斗应当让玩家持续移动
- 玩家不能长期站在原地安全输出
- 敌人必须包含近战、远程、飞行、冲锋和控制类型
- 技能应支持清怪、爆发、位移、控制、召唤等不同作用

## 5. 成长功能需求

### 5.1 局内等级

- 初始等级为1
- 击败敌人掉落经验
- 经验达到阈值后升级
- 升级时暂停战斗
- 每次出现3个升级选项
- 玩家选择1个后继续战斗

### 5.2 升级选项分类

- 新技能
- 技能升级
- 普通攻击强化
- 生存强化
- 特殊机制强化

### 5.3 技能等级

每个主动技能最大5级：

- 1级：解锁
- 2级：基础增强
- 3级：第一次分支
- 4级：强化分支
- 5级：最终进化

## 6. 内容需求

### 6.1 MVP 玩家内容

- 1名主角
- 1把长刀
- 3段普通攻击
- 1种空中攻击
- 1种蓄力攻击
- 3个主动技能
- 1个大招
- 15个以上局内升级项

### 6.2 MVP 敌人内容

- 裂隙杂兵
- 盾甲兵
- 弓箭手
- 自爆怪
- 飞行魔眼
- 冲锋兽
- 1种精英怪
- 1个最终 Boss

### 6.3 MVP 地图

第一张地图：废弃村落。

地图包含：

- 平坦地面
- 两层或三层平台
- 可穿透平台
- 场景边界
- 敌人出生点
- 装饰层
- 远景层
- 特效层

## 7. 用户界面需求

### 7.1 主菜单

- 开始游戏
- 角色
- 技能图鉴
- 天赋
- 设置
- 退出游戏

### 7.2 战斗界面

- 生命值
- 护盾值
- 等级
- 经验条
- 当前时间
- 击杀数
- 连击数
- 三个技能图标
- 大招能量
- Boss 血条
- 暂停按钮

### 7.3 升级界面

每张卡片显示：

- 图标
- 名称
- 当前等级
- 稀有度
- 效果描述
- 数值变化
- 分支标记

### 7.4 结算界面

- 存活时间
- 击杀数量
- 最高连击
- 总伤害
- 受到伤害
- 使用最多的技能
- 获得资源
- 重新开始
- 返回主菜单

## 8. 非功能需求

### 8.1 性能

MVP 目标：

- Windows 中端设备稳定 60 FPS
- 同屏普通敌人 80 个时保持可玩
- 短时间峰值敌人 120 个时不崩溃
- 使用对象池减少频繁创建和销毁

### 8.2 可维护性

- 角色、技能、敌人和升级使用数据配置
- 代码模块化
- 禁止魔法数字散落在脚本中
- 关键参数集中到 Resource 中
- 每个系统具有清晰接口

### 8.3 可扩展性

后续应当可以增加：

- 新角色
- 新武器
- 新技能
- 新地图
- 新敌人
- 新难度
- 新游戏模式

## 9. MVP 不包含内容

- 联机
- PVP
- 大地图探索
- 开放世界
- 复杂剧情树
- 商城
- 抽卡
- 角色换装
- 装备随机词条
- 账号系统
- 云存档

## 10. 验收标准

- 玩家能完整完成一局
- 所有核心输入正常
- 游戏不会因为敌人数量增加而出现严重卡顿
- 技能升级会产生明显视觉和玩法变化
- Boss 拥有至少三个攻击模式
- UI 信息清晰
- 不存在无法继续游戏的流程错误
